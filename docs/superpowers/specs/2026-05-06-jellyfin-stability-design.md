# Jellyfin Stability Hardening ŌĆö Design

- **Date:** 2026-05-06
- **Scope:** `nixos/vm-nix/container/jellyfin.nix`, `nixos/vm-nix/default.nix`
- **Goal:** Eliminate broken systemd dependencies, stop relying on `:latest` images, bound restart loops, and align ports/volumes with how the system actually runs ŌĆö without expanding scope into HW transcoding or migration off Docker.

## Problem Summary

The current Jellyfin module on `vm-nix` (host `192.168.0.240`) was generated by `compose2nix` and has accumulated several stability liabilities:

1. The container declares system-scope `Requires=`/`After=`/`BindsTo=` against unit names such as `home-chin39-mounts-alist.mount`, but those mounts only exist as **user-scope** rclone services (defined in `home-manager/programs/rclone.nix`). System-scope systemd cannot see user units, so the dependencies are inert. When rclone is not yet running, Jellyfin can start with empty bind-mounts and corrupt its library state.
2. The Nix module mounts `/mnt/data/video/jellyfin` (CIFS to `192.168.0.254`), but the colocated `docker-compose.yml` references `/mnt/autofs/data/video/jellyfin` (NFS over WireGuard). The systemd unit also has a stale `RequiresMountsFor=/mnt/autofs/data` that does not match the real volume.
3. The container is built locally from `FROM jellyfin/jellyfin:latest` plus `apt install fonts-noto-cjk-extra`. Every rebuild may pick up a different upstream version and Debian package set, gated on the host HTTP proxy.
4. `Restart=on-failure` has no `RestartSec`, no `StartLimitIntervalSec`, no `StartLimitBurst`. Mount-flap loops can hit systemd's default rate limit and stop trying.
5. There is no Docker `HEALTHCHECK`. A hung FFmpeg or wedged web layer leaves the container "alive but useless" with no signal anywhere.
6. DLNA discovery (`1900`) and Jellyfin client autodiscovery (`7359`) are exposed as TCP, but both protocols are UDP. LAN auto-discovery silently does not work.
7. Two delay timers (`start-jellyfin-stack-delay` boot+2min on the target, `docker-jellyfin-delay` active+120s on the service) layer on top of the broken dependencies. They mask issue 1 with a fixed wait.
8. `virtualisation.docker.autoPrune.enable = true` runs weekly with no filter. After a transient outage, dangling images for the locally-built `compose2nix/jellyfin` could be pruned and force a rebuild on next start.

This design fixes 1ŌĆō8. HW transcoding and migration to native `services.jellyfin` are explicitly out of scope.

## Decisions

Captured during brainstorming on 2026-05-06:

| # | Decision |
|---|---|
| Mount scope | Keep rclone in user-scope; add a system-scope sentinel oneshot that polls the FUSE mounts and Jellyfin `Requires=` it. Enable user lingering for `chin39` so user-scope rclone starts at boot without a session. |
| Media volume | Canonical: `/mnt/data/video/jellyfin` (CIFS LAN to `192.168.0.254`). Drop the stale `RequiresMountsFor=/mnt/autofs/data`. Do **not** set `BindsTo=mnt-data.mount` ŌĆö rely on `Requires=` for startup gating; tolerate brief CIFS blips at runtime. |
| Image build | Pin to a specific upstream tag (e.g. `jellyfin/jellyfin:10.10.x`) and drop the local Dockerfile. Provide CJK fonts via a Nix-managed `symlinkJoin` of `noto-fonts-cjk-sans` + `noto-fonts-cjk-serif` bound read-only at `/config/fonts`. |
| Resource caps | No hard memory/cpu caps. Add a Docker `HEALTHCHECK` (bash TCP builtin) and systemd restart cooldown (`RestartSec`, `StartLimitIntervalSec`, `StartLimitBurst`). |

## Architecture

Three files touched, no new files:

| Path | Change |
|---|---|
| `nixos/vm-nix/container/jellyfin.nix` | Bulk of the work: drop the local Dockerfile build, wire mount sentinel, switch to upstream pinned image, bind-mount CJK fonts from a `symlinkJoin`, fix UDP port mappings, add HEALTHCHECK + restart cooldown, drop both delay timers, replace broken systemd dependencies, scope autoPrune. |
| `nixos/vm-nix/default.nix` | Move `1900` and `7359` from `allowedTCPPorts` to `allowedUDPPorts`. Set `users.users.chin39.linger = true;`. Replace `virtualisation.docker.autoPrune` defaults with a `until=720h` filter. |
| `/home/chin39/Documents/container/jellyfin/build/` | Becomes orphan (Dockerfile no longer consumed). Leave in place. |

### New systemd objects

- `jellyfin-mount-ready.service` ŌĆö Type=oneshot, RemainAfterExit=true, TimeoutStartSec=180. Polls `/home/chin39/mounts/{alist,115-single,union-115}` for both `mountpoint -q` AND non-empty `ls`, with a 120s deadline. Fails on timeout so `Requires=` blocks downstream cleanly.

### Removed systemd objects

- `docker-build-jellyfin.service` ŌĆö local Dockerfile build no longer used.
- `docker-jellyfin-delay.timer` ŌĆö replaced by sentinel correctness gating.
- `start-jellyfin-stack-delay.timer` ŌĆö replaced by sentinel correctness gating.

### Rewired `docker-jellyfin.service`

```nix
systemd.services."docker-jellyfin" = {
  unitConfig = {
    RequiresMountsFor = [ "/mnt/data/video/jellyfin" ];
    StartLimitIntervalSec = "10min";
    StartLimitBurst = 5;
  };
  serviceConfig = {
    Restart = lib.mkOverride 90 "on-failure";
    RestartSec = "10s";
  };
  after = [
    "docker-network-jellyfin_default.service"
    "jellyfin-mount-ready.service"
    "mnt-data.mount"
  ];
  requires = [
    "docker-network-jellyfin_default.service"
    "jellyfin-mount-ready.service"
    "mnt-data.mount"
  ];
  partOf = [ "docker-compose-jellyfin-root.target" ];
};
```

The previous `home-chin39-mounts-*.mount` references and `RequiresMountsFor=/mnt/autofs/data` are deleted. `BindsTo=mnt-data.mount` is **not** added; see "Risks" R1 for the tradeoff.

### Boot activation

With both delay timers removed, the root target activates at boot directly:

```nix
systemd.targets."docker-compose-jellyfin-root" = {
  unitConfig.Description = "Root target generated by compose2nix.";
  wantedBy = [ "multi-user.target" ];
};
```

## Per-Issue Detail

### #1 Mount sentinel + dependency rewiring

```nix
systemd.services."jellyfin-mount-ready" = {
  description = "Wait for user-scope rclone FUSE mounts to be ready";
  path = [ pkgs.coreutils pkgs.util-linux ];
  serviceConfig = {
    Type = "oneshot";
    RemainAfterExit = true;
    TimeoutStartSec = "180";
  };
  script = ''
    set -eu
    deadline=$(( $(date +%s) + 120 ))
    for m in /home/chin39/mounts/{alist,115-single,union-115}; do
      while ! ( mountpoint -q "$m" \
                && [ -n "$(ls -A "$m" 2>/dev/null | head -n1)" ] ); do
        if [ "$(date +%s)" -gt "$deadline" ]; then
          echo "timeout waiting for $m" >&2
          exit 1
        fi
        sleep 2
      done
    done
  '';
};
```

Combined with `users.users.chin39.linger = true;` (in `default.nix`), user-scope rclone starts at boot without an active session, the sentinel observes the mounts populated, and Jellyfin proceeds.

### #2 Volume canonicalization

Container volume already correct. Two cleanups:

- Drop `RequiresMountsFor = [ "/mnt/autofs/data" ];` from the unit.
- Add `RequiresMountsFor = [ "/mnt/data/video/jellyfin" ];` to ensure systemd activates the CIFS automount before Jellyfin reads from it.

`docker-compose.yml` is reference-only on this host; not consumed by NixOS. Optional follow-up: align it with the Nix module to avoid future confusion.

### #3 Pin image, drop local Dockerfile

```nix
let
  jellyfinCjkFonts = pkgs.symlinkJoin {
    name = "jellyfin-cjk-fonts";
    paths = [
      pkgs.noto-fonts-cjk-sans
      pkgs.noto-fonts-cjk-serif
    ];
  };
in
{
  virtualisation.oci-containers.containers."jellyfin" = {
    image = "jellyfin/jellyfin:10.10.7";
    environment = {
      "PGID" = "100";
      "PUID" = "1000";
      "TZ" = "Asia/Shanghai";
      "http_proxy" = "http://192.168.0.240:10809";
      "https_proxy" = "http://192.168.0.240:10809";
    };
    volumes = [
      "/home/chin39/Documents/container/jellyfin/cache:/cache:rw"
      "/home/chin39/Documents/container/jellyfin/config-jellyfin:/config:rw"
      "${jellyfinCjkFonts}/share/fonts/opentype/noto-cjk:/config/fonts:ro"
      "/home/chin39/mounts:/mounts:rw"
      "/mnt/data/video/jellyfin:/jellyfin-media:rw"
    ];
    ports = [
      "8096:8096/tcp"
      "7359:7359/udp"
      "1900:1900/udp"
    ];
    log-driver = "journald";
    extraOptions = [
      "--network-alias=jellyfin"
      "--network=jellyfin_default"
      "--health-cmd=bash -c '</dev/tcp/127.0.0.1/8096'"
      "--health-interval=30s"
      "--health-timeout=5s"
      "--health-retries=3"
      "--health-start-period=60s"
    ];
  };
}
```

Image is pinned to `jellyfin/jellyfin:10.10.7` (latest stable patch on the `10.10.x` line as of 2026-05). Bumping is a one-line tag change followed by `nixos-rebuild switch`. The previous host volume `/home/chin39/Documents/container/jellyfin/fonts:/config/fonts:rw` is removed in favor of the Nix store overlay.

### #4 HEALTHCHECK + restart cooldown

Healthcheck shown above. Probe is the bash TCP builtin (no `curl`/`wget` dependency on the image). It checks liveness only ŌĆö that the listener responds. For the failure modes we care about (FFmpeg hung, web wedged), the listener typically goes silent too.

Restart cooldown shown in the rewired `docker-jellyfin.service` block: `RestartSec=10s`, `StartLimitIntervalSec=10min`, `StartLimitBurst=5`.

### #6 UDP ports

`jellyfin.nix`:

```nix
ports = [
  "8096:8096/tcp"
  "7359:7359/udp"
  "1900:1900/udp"
];
```

`nixos/vm-nix/default.nix` firewall:

- `allowedTCPPorts`: remove `7359`, remove `1900`. `8096` stays.
- `allowedUDPPorts`: add `7359`, add `1900`.

### #7 Collapse delay timers

Delete `systemd.timers."docker-jellyfin-delay"` and `systemd.timers."start-jellyfin-stack-delay"`. Add `wantedBy = [ "multi-user.target" ];` to `docker-compose-jellyfin-root` target so Jellyfin activates at boot via correctness rather than fixed-time delay.

### #8 Auto-prune filter

```nix
virtualisation.docker = {
  enable = true;
  autoPrune = {
    enable = true;
    dates = "weekly";
    flags = [ "--all" "--filter" "until=720h" ];
  };
};
```

`until=720h` means "older than 30 days". The pinned upstream image is pulled regularly enough that it stays out of the prune set under normal operation.

## Apply Order

Each step is a separate commit. Steps land in order so rollback is granular:

1. **Firewall + container ports (#6)** ŌĆö pure config; verifies easily via mobile autodiscovery.
2. **Auto-prune filter (#8)** ŌĆö independent.
3. **Mount sentinel + dep rewiring + delay timer removal + user lingering (#1, #2, #7)** ŌĆö single bundle; tightly coupled.
4. **Image pin + drop build + CJK font join (#3)** ŌĆö switch to upstream tag; remove `docker-build-jellyfin`. Container recreates on `nixos-rebuild switch`.
5. **HEALTHCHECK + restart cooldown (#4)** ŌĆö `extraOptions` and unit-level limits.

Each step is applied via `nixos-rebuild switch`. Container recreates when `image` or `extraOptions` change.

## Verification

| Fix | How to verify |
|---|---|
| #1 sentinel | `systemctl status jellyfin-mount-ready` shows `active (exited)` after boot. Stop a user-scope rclone (`systemctl --user stop rclone-115-single`); restart `docker-jellyfin`; sentinel must time out and block startup, not silently start with empty bind mount. |
| User lingering | `loginctl show-user chin39 \| grep Linger` returns `Linger=yes`. Reboot with no user session active; `systemctl --user --machine=chin39@.host status rclone-alist` shows the unit running. |
| #2 mount dep | `systemd-analyze critical-chain docker-jellyfin.service` shows `mnt-data.mount` and `jellyfin-mount-ready.service` in the chain. |
| #3 image pin | `docker inspect jellyfin --format '{{.Config.Image}}'` returns the pinned tag. `docker images` no longer lists `compose2nix/jellyfin`. CJK subtitle rendering verified by playing a CJK-tagged title. |
| #4 healthcheck | `docker inspect jellyfin --format '{{.State.Health.Status}}'` returns `healthy` after start. `docker pause jellyfin` flips it to `unhealthy` within ~2 minutes. |
| #4 cooldown | Force 5+ rapid failures (e.g., `systemctl stop mnt-data.mount` before `docker-jellyfin` starts). systemd stops retrying after 5/10min per `StartLimitBurst`. `journalctl -u docker-jellyfin -n 200` shows the rate-limit message. |
| #6 UDP | From a phone on LAN: Jellyfin mobile app autodiscovery finds the server (uses `7359/udp`). DLNA-aware client sees it under "Network devices" (`1900/udp`). |
| #7 delays | Cold boot the VM; Jellyfin reaches `active` as soon as sentinel + `mnt-data.mount` resolve, not after a fixed 2-minute wall-clock wait. |
| #8 prune filter | `docker system prune --all --filter until=720h --dry-run` does **not** list the pinned image while the container is running. |

## Risks

- **R1: No `BindsTo=mnt-data.mount`.** Brief CIFS drops do not bounce the container. A sustained CIFS outage leaves Jellyfin alive-but-erroring; the bash-TCP healthcheck stays `healthy` because the listener still responds. Detection during a sustained CIFS outage is therefore manual (logs, user reports). Mitigation: `Requires=mnt-data.mount` still gates startup; restart cooldown bounds any actual crash loop. If this proves wrong in practice, add `BindsTo=mnt-data.mount` and accept the bounce cost.
- **R2 (resolved): user-scope rclone at boot.** `users.users.chin39.linger = true;` makes rclone start without a session. Falls back to `systemd.tmpfiles.rules = [ "f /var/lib/systemd/linger/chin39 0644 root root - -" ];` if the option is unavailable on the running NixOS version.
- **R3: Pinned image tag pulls a buggy upstream version.** Mitigation: pick a known-stable patch tag (latest `10.10.x` rather than a brand-new minor). Rollback is `git revert` of the pin commit + `nixos-rebuild switch`.
- **R4: CJK font coverage.** `symlinkJoin` of `noto-fonts-cjk-sans` + `noto-fonts-cjk-serif` covers the same glyph set as Debian's `fonts-noto-cjk-extra` for typical Chinese/Japanese/Korean subtitles. If a specific title renders boxes, expand the join with additional packages (e.g., `noto-fonts-emoji`) at follow-up time.
- **R5 (resolved): healthcheck dependency.** Bash builtin TCP probe (`bash -c '</dev/tcp/127.0.0.1/8096'`) does not depend on `curl` or `wget` being in the image.

## Out of Scope

- HW transcoding / VAAPI passthrough (`/dev/dri` mapping, AMD VAAPI driver).
- Migration to native NixOS `services.jellyfin`.
- Cleanup of orphan `/home/chin39/Documents/container/jellyfin/build/`.
- Reconciling `docker-compose.yml` with the Nix module (reference-only on this host).
- Multi-source media libraries (NFS-over-WireGuard fallback when off-LAN).
